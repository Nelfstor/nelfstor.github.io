Статический класс

Статический класс - это класс, у которого нет экземпляров, а все его методы и свойства должны быть статические и имеют доступ через имя класса

static class SuperStaticClass

А теперь менее формально: когда вы создаёте обычный класс, и хотите, чтобы у всех его экземляров было общее свойство - вы делаете его статическим. Например, у вас класс message и вы хотите, чтобы у всех message был строго одинаковый заголовок Header. В упрощённом виде это выглядит так: 

class Message
    {
        public static string Header;
        public string Value;
    };

Можно поменять Header, но он разом поменяется для всех экземляров класса. Т.е. можно сказать, что он общий для этого типа.

Теперь представьте, что у вас есть тип, но нет экземляров. Получается, что как-будто бы он - единственный экземпляр. Иногда говорят, что это простейший случай паттерна Singleton.

Экземпляр статического класса не нужно создавать (и нельзя создать целенаправленно).

У такого подхода есть плюсы и минусы:

+ Удобный доступ к синтаксису

- Невозможность создать второй экземпляр. Иногда задача меняется, и даже когда "было точно написано в тех. задании, что класс нужен только один", это может изменится. В случае обычных классов гибкости больше.

- Если класс публичный - то доступ можно получить откуда угодно, а это уменьшает инкапсуляцию, и увеличивает связанность, т.е. кто-то может что-то "случайно" поменять в нашем классе, так как он виден всюду.

Однако это не страшно если класс не хранит состояние (state). Если у класса нет состояния, то его нельзя "случайно изменить", и использование такого класса не считается чем-то плохим. Например, вспомогательные методы часто размещают в статических классах.
