Условная конструкции if-else в C#

  • Главная
  • Описание работы консоли
  • Переменные

В C# существует несколько условных конструкций, сегодня мы сосредоточимся на if-else.

Что вообще означает это словосочетание? – Есть некоторое условие, в зависимости от которого дальнейшее выполнение программы различается.

Представьте себе развилку. Одна дорога ведёт в город, а другая в деревню.

У выбора на перекрёстке есть последствия. В C# их можно сформулировать при помощи конструкции

if (условие верно) {инструкция 1} else {инструкции альтернативного сценария}.

Так, для нашего примера будет:

if (goingToCity == true) // водитель повернул в сторону города
{ 
    ComeToCity(); // Приехать в город
    GoToTheatre();  // Пойти в театр
}
else  // условие ложно
{
    GoToVillage(); // Приехать в деревню
    PlayWithGoose(); // Поиграть с гусём
}

Конструкция с дорогой – весьма наглядная, но подобные условные конструкции, на самом деле, встречаются нам в быту постоянно.

Например, запертая дверь. Если ваш ключ подходит, то вы открываете замок и получаете доступ к содержимому. Если не подходит – не получаете.

if (yourKeyIsOk) // условие имеет значение true или false
{
    YouGetTheAccess(); // Получить доступ
}

В тестах или квизах вам нужно правильно выбрать один вариант из четырёх. Если выбор правильный – вы получаете балл.

if (yourAnswer == rightAnswer) // ваш ответ сравнивается с эталоном
{
    AddOnePoint(); // Получить очко
}

Если вы сели не на свой автобус, то для «альтернативной ветки» появляются свои инструкции:

if (youTookTheRightBus) // вы сели на свой автобус
{
    ContinueYourPlan(); // Благополучное продолжение планов
}
else // в этой ветке вы едете не на том автобусе!
{
    FindTheWayToTourDestination(); // Построить новый маршрут
    SomePeopleTellYouThatYouAreLate(); // Кто-нибудь скажет, что вы опоздали
    YouDoTasksInHurry();  // Придётся спешить 
}

Как-то я опоздал на самолёт в Лондон и мне пришлось лететь через Хельсинки. Альтернативная ветка имела много новых инструкций – мне нужно было купить новый билет, сориентироваться в новом аэропорту, и так далее.

Каждый день мы сталкиваемся с десятками условных конструкций. Ключ и замок в физическом мире аналогичны логину и паролю в приложениях.

Для совершения покупки в интернет-магазинах с вашей карты будет запрошена стоимость покупки. Если средств на вашей карте достаточно, то к вам скорее всего придёт код подтверждения оплаты. Если вы введёте код верно, магазин примет оплату и отправит заказ в исполнение, а вы будете ожидать доставку. Такой сценарий по-прежнему не слишком сложный, но здесь уже две конструкции, одна из них «вложенная».

if (sufficientFunds)
{
    RequestConfirmationCode(); // Запросить код подтверждения
    if (codeIsCorrect)
    {
        AcceptPayment(); // Принять оплату
        SendOrder(); // Отправить заказ
    }
}

Программы во многом состоят из условных конструкций, их совокупность обычно называют «логикой» работы программы.
